# 1. Purpose

Настоящий документ определяет официальный словарь терминов платформы Voxarium.

Все документы спецификации, исходный код, пользовательская документация, комментарии в коде, тестовая документация и API обязаны использовать определения настоящего документа.

Использование иных терминов допускается исключительно при наличии явного отображения на канонический термин.

---

# 2. Terminology Rules

Для каждого понятия существует ровно один канонический термин.

Каждый термин имеет единственное определение.

Использование нескольких терминов для обозначения одной сущности запрещается.

Изменение значения существующего термина запрещается.

При необходимости вводится новый термин.

---

# 3. Canonical Terms

## Project

Основная единица работы пользователя.

Проект объединяет все данные, необходимые для создания речевого контента.

Проект является контейнером верхнего уровня.

---

## Production

Независимый вариант производства аудиоконтента внутри проекта.

Production использует общие исходные данные проекта, но хранит собственные результаты генерации.

Примеры:

• Русская озвучка

• Английская озвучка

• Женская озвучка

• Финальная редакция

---

## Source

Неизменяемое представление импортированных исходных данных.

Source всегда соответствует оригинальному источнику.

---

## Import

Процесс преобразования Source во внутреннее представление платформы.

---

## Document

Каноническое внутреннее текстовое представление.

Document полностью независим от формата импортированного файла.

---

## Timeline

Логическая последовательность воспроизведения.

Timeline определяет порядок генерации и воспроизведения.

---

## Fragment

Минимальная логическая единица озвучивания.

Fragment представляет отдельный участок текста, обладающий собственными параметрами генерации.

---

## Role

Логический персонаж произведения.

Role определяет художественную сущность, но не определяет способ генерации.

---

## Voice Profile

Абстрактное описание желаемого звучания речи.

Voice Profile не является моделью искусственного интеллекта.

---

## Speech Engine

Компонент, выполняющий синтез речи.

Speech Engine всегда реализуется в виде Plugin.

---

## Speech Segment

Результат генерации одного Fragment.

Speech Segment представляет законченный аудиофрагмент.

---

## Audio Track

Последовательность Speech Segment.

Используется для прослушивания и экспорта.

---

## Workflow

Длительный управляемый процесс.

Workflow может включать тысячи отдельных задач.

---

## Job

Минимальная исполняемая задача.

Job является единицей работы планировщика.

---

## Plugin

Независимый расширяемый компонент платформы.

Plugin реализует один или несколько публичных контрактов.

---

## Contract

Публичный интерфейс взаимодействия между Core и Plugin.

Контракт определяет исключительно поведение.

Контракт не содержит реализации.

---

## Module

Компонент Core платформы.

Модуль обладает собственной областью ответственности.

---

## Command

Намерение изменить состояние системы.

Команда инициирует выполнение бизнес-операции.

---

## Event

Сообщение о факте уже произошедшего изменения.

Событие является неизменяемым.

---

## Query

Запрос данных.

Query не изменяет состояние системы.

---

## Message Bus

Инфраструктурный компонент доставки Commands, Events и Queries.

---

## Worker

Исполнитель фоновых задач.

Worker выполняет Job.

---

## Asset

Любой бинарный объект проекта.

---

## Cache

Временные данные, подлежащие автоматическому восстановлению.

---

## Preset

Сохранённый набор пользовательских параметров.

---

## Export Profile

Сохранённая конфигурация процесса экспорта.

---

## Plugin Data

Данные, принадлежащие конкретному Plugin.

Core не анализирует их внутреннюю структуру.

---

## Core

Минимальная неизменяемая часть платформы.

Core предоставляет инфраструктуру выполнения и публичные контракты.

---

## SDK

Набор средств разработки Plugin.

SDK включает библиотеки, интерфейсы, шаблоны проектов и средства тестирования.

---

Запрещается использовать следующие обозначения вместо канонических терминов.

Workspace → Project

Session → Project

Book → Project

Voice Model → Voice Profile

Scene → Fragment

Task → Job

Process → Workflow

Extension → Plugin

Interface → Contract

Engine Module → Speech Engine

# 6. Language Rules

Каноническим языком архитектуры является английский язык.

Русскоязычная документация обязана использовать официальные английские термины без перевода при упоминании архитектурных сущностей.

Например:

Project

Timeline

Fragment

Workflow

Voice Profile

Plugin

Command

Event

---

# 7. Naming Rules

Все названия сущностей в исходном коде обязаны совпадать с каноническими терминами настоящего документа.

Сокращения допускаются исключительно для общеупотребительных терминов.

Например:

UUID

API

SDK

TTS

GUI

Не допускается создание собственных сокращений.

# 8. Compliance

Использование терминологии, противоречащей настоящему документу, считается нарушением архитектурной спецификации.
